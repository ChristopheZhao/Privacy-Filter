# Privacy Filter

A desktop app for removing sensitive information from text while preserving readable context. It is designed for prompt sanitization, log sharing, and document cleanup.

## Features

- Built-in rules for common sensitive data
  - Chinese ID cards
  - Phone numbers
  - Bank cards
  - Email addresses
  - Social accounts
  - API keys and credentials
  - Database connection strings
  - IP addresses
- Custom regex rules
- Local-only processing
- Deep filtering with a local LLM
- Desktop packaging with Tauri

## Local LLM Strategy

The product should support multiple local provider choices over time.

- Benchmarking is used to choose a sensible default model and deep-filter strategy, not to hard-code one forever.
- The app configuration exposes `provider`, `base_url`, `model`, `timeout`, and related local-LLM settings.
- The current first working provider is `ollama`.

Current benchmark-backed recommendation on the `60`-sample starter corpus:

- Default deep-filter path: `pure + qwen3.5:4b + priors-v2`
- Current model recommendation: `qwen3.5:4b`
- Lower-cost fallback candidate: `qwen3.5:2b`, with clearly lower recall than `4b`
- Regex remains in the product for quick preview and fallback, but it is no longer the preferred deep-filter strategy direction.

Current caveat:

- The current decision is based on a `60`-sample benchmark pause point. It is strong enough to choose the next implementation direction, but not yet large enough to claim final release-grade coverage across all privacy categories.

## Development

### Prerequisites

- Node.js 18+
- Rust toolchain
- Tauri build dependencies for your platform

For WSL/Linux development you also need the usual Tauri GUI dependencies, such as GTK/WebKit-related packages.

### Install Dependencies

```bash
npm install
```

### Start Web Development Server

```bash
npm run dev
```

### Start Desktop App in Development

```bash
npm run dev:tauri
```

### Build Web Assets

```bash
npm run build
```

### Build Desktop Installer

```bash
npm run build:tauri
```

### Run Local LLM Benchmark

```bash
npm run bench:local-llm-filter
```

## Configuration Storage

The desktop app stores rule state and local-LLM settings through Tauri commands:

- `load_config`
- `save_config`
- `llm_healthcheck`
- `analyze_text`

These settings are written to the platform-specific app config directory.

## Local LLM Prerequisites

The current implementation assumes a local provider endpoint, with `ollama` as the first supported provider.

Typical local setup:

```bash
ollama pull qwen3.5:4b
ollama serve
```

Default endpoint:

```text
http://127.0.0.1:11434
```

## Language Support

The built-in rules and starter benchmark corpus currently focus on Chinese-first scenarios. If you need support for other regions, you can add custom rules in the UI or extend the built-in rule set in:

- `src/components/TextFilter/privacyRules.js`

## Notes

- All filtering is performed locally.
- "深度过滤" in the UI means local-LLM-based filtering.
- Packaged builds are generated by Tauri.
- If you develop inside WSL, make sure your GUI environment and desktop libraries are available before running `npm run dev:tauri`.
