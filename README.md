# Privacy Filter

A desktop app for removing sensitive information from text while preserving readable context. It is designed for prompt sanitization, log sharing, and document cleanup.

## Features

- Built-in rules for common sensitive data:
  - Chinese ID cards
  - Phone numbers
  - Bank cards
  - Email addresses
  - Social accounts
  - API keys and credentials
  - Database connection strings
  - IP addresses
  - Contract numbers
- Custom regex rules
- Local-only processing
- Import and export for custom rules
- Desktop packaging with Tauri

## Tech Stack

- React 18
- Vite 6
- Tailwind CSS
- Tauri 2
- Rust

## Development

### Prerequisites

- Node.js 18+
- Rust toolchain
- Tauri build dependencies for your platform

For WSL/Linux development you also need the usual Tauri GUI dependencies, such as GTK/WebKit-related packages.

### Install Dependencies

```bash
npm install
```

### Start Web Development Server

```bash
npm run dev
```

### Start Desktop App in Development

```bash
npm run dev:tauri
```

### Build Web Assets

```bash
npm run build
```

### Build Desktop Installer

```bash
npm run build:tauri
```

## Configuration Storage

The desktop app stores custom rules and active rule state through Tauri commands:

- `load_config`
- `save_config`

These settings are written to the platform-specific app config directory.

## Language Support

The built-in rules currently focus on Chinese text scenarios. If you need support for other regions, you can add custom rules in the UI or extend the built-in rule set in:

`src/components/TextFilter/privacyRules.jsx`

## Notes

- All filtering is performed locally.
- Packaged builds are generated by Tauri.
- If you develop inside WSL, make sure your GUI environment and desktop libraries are available before running `npm run dev:tauri`.
